Below is a draft **JSON Schema (Draft 2020-12)** for the AIARD (Abductive Intelligence Analytical Reasoning Database) domain model. Rather than mirroring every SQL table 1:1, this schema represents the **canonical exchange format** that applications, APIs, ETL pipelines, LLM agents, and ML workflows can use.

# Top-Level Investigation Schema

```json
{

  "$id": "https://aiard.org/schema/investigation.json",
  "title": "AIARD Investigation",

    "leadAnalyst": {

    },

      "type": "array",

        "MODERATE",
        "HIGH"
      ]

        },

    "Source": {

      ],

        "observationId": {

          "type": "string"

        }
      }

    "Hypothesis": {

      "required": [
        "hypothesisId",

      ],

          "$ref": "#/$defs/Likelihood"

          "type": "boolean"
        }
      }
    },

    "Indicator": {
      "type": "object",
      "required": [
        "indicatorText"
      ],
      "properties": {
        "indicatorId": {
          "type": "string"
        },
        "indicatorText": {
          "type": "string"
        },

          "type": "boolean"

        "expectedBy": {
          "type": "string",

    "Recommendation": {
      "type": "object",
      "properties": {

          "type": "string"
```

# ML-Friendly Training Record

A normalized record for LLM fine-tuning, forecasting, and confidence calibration:

```json
{
  "case_id": "INV-2026-001",
  "observations": [
    "New domain registered",
    "Infrastructure overlaps known actor"
  ],
  "entities": [
    "Actor-X",
    "Domain-123"
  ],
  "hypotheses": [
    {
      "id": "H1",
      "text": "Actor-X preparing operation",
      "likelihood": "LIKELY",
      "confidence": "MODERATE"
    },
    {
      "id": "H2",
      "text": "Benign activity",
      "likelihood": "POSSIBLE",
      "confidence": "LOW"
    }
  ],
  "selected_hypothesis": "H1",
  "decision": "Increase monitoring",
  "outcome": "Confirmed hostile activity"
}
```

This structure provides a foundation for:

* Relational storage
* API exchange
* Knowledge graph generation
* RAG indexing
* Entity resolution
* Hypothesis ranking models
* Confidence calibration models
* Forecasting and decision-support agents
* LLM reasoning trace training datasets
