A production-grade PostgreSQL schema for an **Abductive Intelligence & Analytical Reasoning Database (AIARD)** designed to support:

* Observations (facts)
* Sources
* Investigations
* Alternative hypotheses
* Evidence mapping
* Assumptions
* Confidence assessments
* Indicators/predictions
* Decisions
* Outcomes
* Analyst notes
* Auditability

This schema is normalized and designed to preserve the full reasoning chain.

***

# Users / Analysts

```sql
CREATE TABLE analysts (
    analyst_id BIGSERIAL PRIMARY KEY,
    username VARCHAR(100) UNIQUE NOT NULL,
    full_name VARCHAR(200),
    email VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

# Investigations

```sql
CREATE TABLE investigations (
    investigation_id BIGSERIAL PRIMARY KEY,

    title VARCHAR(300) NOT NULL,
    description TEXT,

    domain VARCHAR(100),
    status VARCHAR(50),

    priority VARCHAR(25),

    lead_analyst_id BIGINT,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    closed_at TIMESTAMP,

    FOREIGN KEY (lead_analyst_id)
        REFERENCES analysts(analyst_id)
```

# Sources

```sql
CREATE TABLE sources (

    source_type VARCHAR(100),

    reliability_rating INTEGER
```

# Observations

Observations are facts.

```sql
CREATE TABLE observations (
    observation_id BIGSERIAL PRIMARY KEY,

    investigation_id BIGINT NOT NULL,

    source_id BIGINT,

    observation_time TIMESTAMP,

    observation_text TEXT NOT NULL,

    confidence_code VARCHAR(20),

    recorded_by BIGINT,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (investigation_id)
        REFERENCES investigations(investigation_id),

    FOREIGN KEY (source_id)
        REFERENCES sources(source_id),

    FOREIGN KEY (recorded_by)
        REFERENCES analysts(analyst_id),

    FOREIGN KEY (confidence_code)
        REFERENCES confidence_lookup(confidence_code)
);
```

# Hypotheses

```sql
CREATE TABLE hypotheses (
    hypothesis_id BIGSERIAL PRIMARY KEY,

    investigation_id BIGINT NOT NULL,

    hypothesis_title VARCHAR(300),

    hypothesis_text TEXT NOT NULL,

    likelihood_code VARCHAR(30),

    confidence_code VARCHAR(20),

    status VARCHAR(50),

    created_by BIGINT,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (investigation_id)
        REFERENCES investigations(investigation_id),

    FOREIGN KEY (created_by)
        REFERENCES analysts(analyst_id),

    FOREIGN KEY (likelihood_code)
        REFERENCES likelihood_lookup(likelihood_code),

    FOREIGN KEY (confidence_code)
        REFERENCES confidence_lookup(confidence_code)
);
```

Example:

```text
H1 - Actor X Controls Infrastructure

H2 - Infrastructure Shared by Multiple Actors

H3 - Benign Research Activity
```

# Evidence Mapping

Links facts to theories.

```sql
CREATE TABLE evidence_links (
    evidence_link_id BIGSERIAL PRIMARY KEY,

    hypothesis_id BIGINT NOT NULL,

    observation_id BIGINT NOT NULL,

    relationship_code VARCHAR(20),

    strength_score INTEGER
        CHECK (strength_score BETWEEN 1 AND 10),

    analyst_notes TEXT,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (hypothesis_id)
        REFERENCES hypotheses(hypothesis_id),

    FOREIGN KEY (observation_id)
        REFERENCES observations(observation_id),

    FOREIGN KEY (relationship_code)
        REFERENCES relationship_lookup(relationship_code)
);
```

# Assessment View

Useful for executive dashboards.

```sql
CREATE VIEW investigation_assessments AS
SELECT

    i.investigation_id,
    i.title,

    h.hypothesis_id,
    h.hypothesis_title,

    h.likelihood_code,

    h.confidence_code,

    COUNT(DISTINCT e.evidence_link_id)
        AS evidence_count,

    COUNT(DISTINCT a.assumption_id)
        AS assumption_count,

    COUNT(DISTINCT ind.indicator_id)
        AS indicator_count

FROM investigations i

JOIN hypotheses h
    ON i.investigation_id = h.investigation_id

LEFT JOIN evidence_links e
    ON h.hypothesis_id = e.hypothesis_id

LEFT JOIN assumptions a
    ON h.hypothesis_id = a.hypothesis_id

LEFT JOIN indicators ind
    ON h.hypothesis_id = ind.hypothesis_id

GROUP BY
    i.investigation_id,
    i.title,
    h.hypothesis_id,
    h.hypothesis_title,
    h.likelihood_code,
    h.confidence_code;
```
