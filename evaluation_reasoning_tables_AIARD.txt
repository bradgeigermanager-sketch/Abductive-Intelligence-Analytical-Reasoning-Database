These are the tables that turn the system into a structured intelligence reasoning platform with an attached investigation database.

The key shift is:

```text
Observations → Information

Hypotheses → Explanations

Requirements → Questions

Gaps → Unknowns

Bias Checks → Reasoning Quality

Decisions → Action

Outcomes → Learning
```

***

# Priority Intelligence Requirements (PIRs)

Tracks the questions leadership actually wants answered.

```sql
CREATE TABLE priority_intelligence_requirements (

    pir_id BIGSERIAL PRIMARY KEY,

    investigation_id BIGINT,

    pir_code VARCHAR(100),

    title VARCHAR(500),

    requirement_text TEXT NOT NULL,

    priority_level VARCHAR(20),

    requester_entity_id BIGINT,

    status VARCHAR(50),

    due_date DATE,

    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY(investigation_id)
        REFERENCES investigations(investigation_id),

    FOREIGN KEY(requester_entity_id)
        REFERENCES entities(entity_id)
);
```

Example:

```text
PIR-001

Will Actor A conduct ransomware operations
against healthcare organizations in the next 90 days?
```

# Key Intelligence Questions (KIQs)

Questions supporting a PIR.

```sql
CREATE TABLE key_intelligence_questions (

    kiq_id BIGSERIAL PRIMARY KEY,

    pir_id BIGINT NOT NULL,

    question_text TEXT NOT NULL,

    priority_level VARCHAR(20),

    status VARCHAR(50),

    FOREIGN KEY(pir_id)
        REFERENCES priority_intelligence_requirements(pir_id)
);
```

Example:

```text
Has Actor A acquired new infrastructure?

Has Actor A modified targeting?
```

# ACH Matrix

Evidence vs. hypothesis scoring.

```sql
CREATE TABLE ach_matrix (

    ach_matrix_id BIGSERIAL PRIMARY KEY,

    ach_id BIGINT,

    hypothesis_id BIGINT,

    observation_id BIGINT,

    consistency_score INTEGER,

    comments TEXT,

    FOREIGN KEY(ach_id)
        REFERENCES competing_hypotheses(ach_id),

    FOREIGN KEY(hypothesis_id)
        REFERENCES hypotheses(hypothesis_id),

    FOREIGN KEY(observation_id)
        REFERENCES observations(observation_id)
);
```

Suggested scale:

```text
+2 Strongly Consistent
+1 Consistent
 0 Neutral
-1 Inconsistent
-2 Strongly Inconsistent
```

# Final Maturity Model

With all components, the architecture becomes:

```text
Entity Layer
│
├── Persons
├── Organizations
├── Infrastructure
├── Campaigns
├── Events
└── Technical Objects

Observation Layer
│
├── Sources
├── Observations
├── Claims
└── Evidence

Reasoning Layer
│
├── Hypotheses
├── ACH
├── Assumptions
├── Indicators
├── Alternatives
└── Bias Reviews

Decision Layer
│
├── Requirements
├── Recommendations
├── Decisions
└── Outcomes

Learning Layer
│
├── Confidence History
├── Outcome Evaluation
├── Lessons Learned
└── Assumption Validation
```

At this point, the database captures not just *what analysts concluded*, but *how they reasoned, what they didn't know, what alternatives they considered, how confident they were, what actions were taken, and whether those actions proved correct*. That's the foundation of a true abductive-reasoning intelligence platform.
