Moving from an **investigation database** to a **knowledge-reasoning ontology**.

The design captures:

```text
Observation
→ Evidence
→ Hypothesis
→ Decision
→ Outcome
```

And mature analytical systems also need to capture:

```text
General Pattern
→ Activities
→ Conditions
→ Indicators
→ Observations
→ Cases
→ Specific Instances
```

This allows the system to learn from many investigations and build reusable knowledge.

***

# Layer 1: Knowledge Objects

Create a supertype for reusable knowledge.

```sql
CREATE TABLE knowledge_objects (

    knowledge_object_id BIGSERIAL PRIMARY KEY,

    object_type VARCHAR(100),

    title VARCHAR(500),

    description TEXT,

    confidence_score NUMERIC(5,2),

    created_at TIMESTAMP,

    updated_at TIMESTAMP
);
```

Types:

```text
PATTERN
CONDITION
ACTIVITY
INDICATOR
RULE
MODEL
HEURISTIC
FRAMEWORK
TACTIC
TECHNIQUE
PROCEDURE
```

***

# Layer 2: Activity Library

Activities become generalized behaviors.

```sql
CREATE TABLE activities (

    knowledge_object_id BIGINT PRIMARY KEY,

    activity_category VARCHAR(200),

    activity_name VARCHAR(300),

    objective TEXT,

    typical_duration VARCHAR(100),

    FOREIGN KEY (knowledge_object_id)
        REFERENCES knowledge_objects(knowledge_object_id)
);
```

Examples:

```text
Credential Harvesting

Reconnaissance

Infrastructure Staging

Narrative Amplification

Financial Laundering

Recruitment
```

# Layer 13: Abstract State Models

A powerful concept.

```sql
CREATE TABLE state_models (

    model_id BIGSERIAL PRIMARY KEY,

    model_name VARCHAR(300),

    description TEXT
);
```

```sql
CREATE TABLE states (

    state_id BIGSERIAL PRIMARY KEY,

    model_id BIGINT,

    state_name VARCHAR(200),

    FOREIGN KEY(model_id)
        REFERENCES state_models(model_id)
);
```

```sql
CREATE TABLE state_transitions (

    transition_id BIGSERIAL PRIMARY KEY,

    source_state_id BIGINT,

    target_state_id BIGINT,

    trigger_condition TEXT,

    probability NUMERIC(5,2),

    FOREIGN KEY(source_state_id)
        REFERENCES states(state_id),

    FOREIGN KEY(target_state_id)
        REFERENCES states(state_id)
);
```

Example:

```text
Normal
→ Suspicious
→ Active Threat
→ Confirmed Incident
→ Remediation
```

# Ultimate Architecture

The complete system becomes:

```text
Reality Layer
├── Entities
├── Observations
├── Events
└── Claims

Reasoning Layer
├── Hypotheses
├── Assumptions
├── ACH
├── Confidence
└── Decisions

Knowledge Layer
├── Activities
├── Conditions
├── Patterns
├── Indicators
├── Rules
├── Models
└── Taxonomies

Learning Layer
├── Outcomes
├── Evaluations
├── Lessons Learned
└── Confidence Calibration

Prediction Layer
├── States
├── Transitions
├── Indicators
└── Forecasts
```

This transforms the design into a **knowledge-centric abductive reasoning platform** capable of not only recording specific cases, but also accumulating generalized knowledge about activities, conditions, patterns, causal relationships, and state transitions that can be reused across future investigations.
